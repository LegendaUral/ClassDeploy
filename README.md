# ClassDeploy — управление учебным классом

Инструмент дистанционного управления учебными ПК: трансляция экранов, установка ПО, блокировка, управление мышью, звук и аудиоконтент.

## Структура проекта

```
ClassDeploy/
├── shared/                       # Общий код: протокол, конфигурация
├── server/                       # Сервер (PyQt6 GUI + WebSocket)
├── agent/                        # Агент (интерактивный запуск из реестра)
├── start_server.bat              # Запуск сервера
├── build_agent.bat               # Сборка EXE агента (единожды)
├── install_agent.bat             # Установка агента на ПК ученика
├── uninstall_agent.bat           # Удаление агента
├── prepare_offline_packages.bat  # Подготовка оффлайн-зависимостей
└── requirements.txt
```

## Быстрый старт

### 1. Сервер (ПК учителя)

```bat
pip install -r requirements.txt
start_server.bat
```

### 2. Агент (ПК ученика) — современный поток установки

**Шаг 1.** На ПК учителя соберите EXE:
```bat
build_agent.bat
```

**Шаг 2.** Скопируйте `build/ClassDeployAgent/ClassDeployAgent.exe` на ПК ученика.

**Шаг 3.** На ПК ученика запустите установщик от администратора:
```bat
install_agent.bat
```
Введите IP адрес ПК учителя.

**Как это работает:**
- Агент устанавливается в `C:\ProgramData\ClassDeploy\Agent`
- Запуск через реестр (HKLM Run) — активен для **каждого вошедшего пользователя**
- Дополнительная строка запуска в стартовой папке для надёжности
- При крахе процесс автоматически перезапускается (watchdog)

### 3. Поиск адреса сервера

Агент ищет сервер в следующем порядке:
1. Переменная окружения `CLASS_DEPLOY_SERVER`
2. Файл `C:\ProgramData\ClassDeploy\server.txt` (создаётся при установке)
3. `127.0.0.1` (локальный тест)

### 4. Логирование

- Лог агента: `C:\ProgramData\ClassDeploy\logs\agent.log`
- **Fallback:** если папка логов недоступна, агент пишет в `LOCALAPPDATA` или `TEMP`
- Сессионный лог: per-session mutex предотвращает конфликты между пользователями

### 5. Удаление агента

```bat
uninstall_agent.bat
```
или вручную:
```bat
sc stop ClassDeployAgent
sc delete ClassDeployAgent
reg delete HKLM\Software\Microsoft\Windows\CurrentVersion\Run /v ClassDeployAgent /f
```

## Возможности

| Функция | Описание |
|---|---|
| 📦 Установка ПО | Drag-and-drop `.msi`/`.exe`/`.zip`, автокликер для мастеров установки |
| 🖥 Трансляция экранов | Сетка экранов, двойной клик — полный экран с масштабированием |
| 🖱 Управление (live) | В полноэкранной трансляции: **только управление мышью** (кнопки, движение, прокрутка) |
| 🔒 Блокировка | Полный экран с чёрным оверлеем и сообщением, невозможно закрыть |
| 😱 Скример | Полноэкранное изображение (своя картинка) на всех или выбранных ПК |
| 💬 Сообщения | Всплывающее окно (MessageBox) на ПК ученика |
| 🔊 Звук | Управление громкостью, мут/анмут локального микса |
| 🎵 Аудиотрансляция | Проигрывание аудиофайла на выбранных ПК (play/stop) |
| ⌨ Клавиши | Win+D, Ctrl+Alt+Del, Alt+F4, Ctrl+Shift+Esc и др. |
| 📵 Ограничения | Блокировка доступа к сайтам и приложениям |
| ⏻ Управление питанием | Выключение, перезагрузка, выход из сеанса |
| 🔍 Процессы | Диспетчер задач: список, имена, пути, поиск, завершение |

## Технические особенности

- **Per-session мьютекс:** каждый вошедший пользователь запускает свой экземпляр агента
- **Интерактивный watchdog:** перезагрузка агента после аварийного завершения
- **Fallback логирования:** если основной путь недоступен, лог идёт в TEMP
- **Переподключение:** агент периодически проверяет `server.txt` и переподключается при смене IP
- **Управление только мышью:** в режиме полного экрана нет захвата клавиатуры (безопасность)

## WebSocket порт

**8765** (TCP) — убедитесь, что фаервол позволяет входящие соединения на ПК учителя.

## Зависимости

- Python 3.10+
- **Кроссплатформенные:** `websockets`, `PyQt6`, `mss`, `Pillow`
- **Windows только:** `pywin32`, `pywinauto`, `pycaw`, `pynput`, `pyinstaller`

Установка: `pip install -r requirements.txt`

## Оффлайн-зависимости

Для машин без интернета используйте:

```bat
prepare_offline_packages.bat    # На ПК с интернетом
```

Это скачает все wheel-файлы в папку `vendor/wheels`. Распакуйте эту папку на офлайн-машину и используйте:

```bat
pip install -r requirements.txt --find-links vendor/wheels --no-index
```
